I create websites using HTML, CSS, and JavaScript. Using WordPress as the main CMS enables my clients to take charge of content management once their project is complete – if they so wish. I additionally focus on responsive design to ensure my client’s sites work well on any screen size and device. I also offer copywriting, proofreading and editing as well as content management services.

My role in the creation of your web presence can be anything from a one-time meeting to an ongoing experience with continuous support.

About

I am a linguist turned front-end web developer. I have been fascinated by language for as long as I can remember. After completing my graduate studies in linguistics and communications, I worked as a remedial language therapist, and communications instructor. When I started playing around with code at one of my favourite organizations, Ladies Learning Code, I immediately fell in love with its possibilities. Now I get to combine two of my favourite things: arts & crafts and language. I am eager to learn new things every day and taking on challenging projects excites and motivates me. I work remotely, but am currently located in Toronto, Ontario. If you have a project you would like to chat about, please contact me below or send me an email to hello@annacolebatch.com

Continous Deployment:

Maturity Model Phase 1(3-6 months)

  1. Trusted Source Control and Issue tracking
  2. TDD
  3. Continous integration
  4. Automation of deployments
  5. And you are still nowhere near continous deployment

Phase 1 Activities:

Write test first Write deployment scripts first Pair program on the challenging parts Increase the frequency of everything which means automate everything

TDD in 1 Week Outline: 1. Refuse to write console.log statements or use the debugger(because of typo problem such as semi-colon or type checking problem) 2. If you are using Javascript,you are screwed if you dont write tests.

Automation in 1 Week 1. Dont type anything in a remote shell.Do eveything via Fabric,Capistrano,etc 2. Put your automation code in git submodule so you can reuse across your projects 3. When stable(after a few projects) turn it into a node module with reference in package.json

Maturity Model Phase 2(6-9 months):

What are the chances of getting a small feature deployed into production safely in 1 hr

  1. Feature based thinking A. small,logical commits with good commit messages B. good tests(and an architect that supports them) C. Continous integration metrics are trending excellent

Comfortable feature flow 1. WIP is managed via real Scrum/Kanban 2. all features reviewed or pair-programmed(strongly related to pull request) 3. deployment to staging is semi-automated 4. Focus on clarity,sustainability,and velocity - i.e flow,not capacity

Phase 2 Activities:

  1. Lots of Git hooks
  2. JSHint
  3. Reject any commit that doesn't pass lint and that doesn't have a headline
  4. Single commit per pull request(git inteactive rebasing)
  5. Everyone is a Git ninja and the commits are clear,concise and independent

Phase 3(9-12 months):

Are you ready for multiple deploys per day? 1. First commit acceptance rate 2. % Issues captured on staging versus production 3. Are you discussing getting rid of the staging server 4. How fast are your deploys?Rollbacks?

Phase 3 Activities:

  1. Review performance metrics
  2. Clear define the cost-benefit of production failures
  3. Increase frequencies of deployments in the concept of your production failure risk profile
